## Decision Rules

### Scenario 1: Table A1 explicitly states beta_d = 4.5 and beta_q = 18.0

**Condition:** Both values match NotebookLM report exactly (within rounding).

**If values are stated as FITTED to H(z) data:**

**Provenance update:**
```python
# In src/beta_provenance.py:

"beta_d_A1": BetaProvenance(
    provenance_status="manuscript_reported_fitted",
    source_type="manuscript_appendix",
    first_known_appearance="[Manuscript title], Table A1, [arXiv:XXXX.XXXXX], page [XX]",
    use_permission_status="allowed_for_fit_reproduction_only",
    evidence_trail="✅ VERIFIED: [Exact quote from table]. Context: Fitted to [dataset]. [Table caption].",
)
```

**Use permission:**
- `allowed_for_fit_reproduction_only` — can reproduce the fit, NOT predict new data
- H(z) modeling **REMAINS BLOCKED** for predictive claims (circular reasoning)
- Can ONLY compare fit quality with ΛCDM on SAME dataset

**Next step:** Create `docs/18_fit_reproduction_requirements.md` documenting:
- Exact H(z) functional form used
- Dataset specification (z ranges, data points, uncertainties)
- Objective function (chi-squared? other?)
- Fixed parameters (Ω_m, H_0, etc.)
- Minimization procedure

**If values are stated as DERIVED from IDM/MULTING theory:**

**Provenance update:**
```python
"beta_d_A1": BetaProvenance(
    provenance_status="manuscript_reported_derived",
    source_type="manuscript_appendix",
    first_known_appearance="[Manuscript title], Table A1, [arXiv:XXXX.XXXXX], page [XX]",
    use_permission_status="source_confirmed",
    evidence_trail="✅ VERIFIED: [Exact quote]. Context: Derived from [formula/equation]. [Table caption].",
)
```

**Use permission:**
- `source_confirmed` — OK to use for H(z) predictions
- H(z) modeling **UNLOCKED** (first source-confirmed beta values)

**Next step:** Implement H(z) solver and test against observations

### Scenario 2: Table A1 does NOT contain beta_d = 4.5 or beta_q = 18.0

**Condition:** Table A1 not found OR does not contain these values.

**Provenance update:**
```python

    evidence_trail="❌ HALLUCINATION CONFIRMED: Manual verification failed. Table A1 not found OR beta_d=4.5 not present in table. NotebookLM error documented.",
)
```

**Action:**
1. Mark NotebookLM candidates as hallucinated
2. Delete or archive beta_d_A1 and beta_q_A1 from registry (or keep with hallucination status for documentation)
3. Prepare clarification email to Dr. Buckholtz (see docs/12_beta_clarification_brief.md)

**Next step:** Email Dr. Buckholtz with Primary Question about beta provenance

### Scenario 3: Table A1 contains DIFFERENT values

**Condition:** Table A1 exists and reports beta values, but NOT 4.5 and 18.0.

**Example:** Table A1 reports beta_d = 4.2, beta_q = 17.5 (different from NotebookLM)

**Action:**
1. Mark NotebookLM candidates as `hallucinated_or_misread`
2. Add NEW beta records with ACTUAL values from Table A1:
   ```python
   "beta_d_A1_actual": BetaProvenance(
       beta_name="beta_d_A1_actual",
       value=4.2,  # Actual value from table
       provenance_status="manuscript_reported_fitted",  # or _derived, depending on context
       ...
   )
   ```
3. Document discrepancy: "NotebookLM reported beta_d=4.5, actual manuscript value is beta_d=4.2"

**Next step:** Use ACTUAL values from manuscript, ignore NotebookLM

### Scenario 4: Table A1 found, but values are ASSUMED or ILLUSTRATIVE

**Condition:** Table A1 states beta_d=4.5, beta_q=18.0 but context says "example values" or "assumed for illustration".

**Provenance update:**
```python

    use_permission_status="allowed_for_toy_model_only",
    evidence_trail="✅ VERIFIED in manuscript, BUT stated as assumed/illustrative values. Context: [quote]. NOT fitted or derived.",
)
```

**Use permission:**
- `allowed_for_toy_model_only` — can explore, NOT for claims
- H(z) modeling **REMAINS BLOCKED**
